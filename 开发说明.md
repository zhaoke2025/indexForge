# IndexForge 开发说明

## 1. 项目目标

开发一个 Web 端后台生成器，用于根据母版 `index.html`、软件名称、版本号、25 个维度配置、9 条硬性要求，自动生成新的 `index.html`。

生成结果必须支持预览、重新生成、复制源码和下载。

## 2. 输入文件

### 2.1 母版-index.html

作为生成新首页的基础母版。生成器必须基于它做安全替换，不能完全重写。

母版用于承载正式上线所需的页面骨架和约束，例如 Tailwind 显示配置、菜单展开方式、iframe 加载、跨 iframe 导航、message 监听等。上传或内置母版的目的，是让最终生成的 `index.html` 可以按母版结构集成到业务系统中。

### 2.2 维度配置说明.txt

第一组配置，包含 25 个可调维度。

这些维度用于控制页面风格，例如配色、圆角、字体、明暗模式、阴影、侧边栏宽度、菜单激活态、用户信息等。

### 2.3 硬性要求.txt

第二组规则，包含 9 条强制要求。

这些要求用于校验生成结果，校验失败不得下载。

### 2.4 examples 参考图

DeepSeek 生成过的页面效果图，只作为视觉参考，不作为核心逻辑依据。

## 3. 功能模块

### 3.1 首页生成

- 输入软件名称。
- 输入版本号，默认 `V1.0`。
- 如果软件名称已包含版本号，则不重复追加。
- 如果软件名称不含版本号，则自动追加 `V1.0`。
- 支持根据软件名称调用大模型生成菜单模块。
- 选择风格预设。
- 配置右上角用户名称、角色、头像图标、下拉菜单。
- 点击生成。
- 生成后运行校验。
- 校验通过后允许预览、复制和下载。

### 3.2 维度配置

展示并管理 25 个维度。

每个维度建议包含：

```ts
type DimensionConfig = {
  id: string;
  name: string;
  group: string;
  enabled: boolean;
  description: string;
  value: any;
  options?: any[];
};
```

### 3.3 要求配置

展示并管理 9 条要求。强制项默认锁定，不允许删除。

```ts
type RequirementRule = {
  id: string;
  name: string;
  level: 'required' | 'recommended' | 'flexible';
  enabled: boolean;
  locked: boolean;
  description: string;
  validateType: string;
};
```

### 3.4 母版管理

- 内置默认母版。
- 支持上传新的 `index.html`。
- 支持源码查看。
- 支持母版结构检测。
- 检测通过后才允许设为默认母版。

检测项：

- `tailwind.config`
- `<h1>`
- `iframe#contentFrame`
- `menuConfig`
- `renderPrimaryMenu`
- `renderSubMenuInPlace`
- `navigateToSub`
- `loadPage`
- `window.navigateTo`
- `message` 监听
- `insertAdjacentElement('afterend', wrapper)`

### 3.5 生成记录

- 软件名称
- 版本号
- 风格预设
- 生成时间
- 生成源码
- 重新预览
- 重新下载

## 4. 第一版开发范围

第一版先实现 MVP，不要一次做太复杂。

必须完成：

1. Web 前端项目初始化。
2. 输入软件名称。
3. 自动版本号 `V1.0`。
4. 选择 8 个风格之一。
5. 配置右上角用户名。
6. 基于母版生成 `index.html`。
7. 校验 9 条硬性要求。
8. iframe 预览。
9. 复制源码。
10. 下载 `index.html`。

第二版再扩展：

1. 25 个维度逐项编辑。
2. 母版上传。
3. 历史记录。
4. 菜单结构自定义。
5. AI 根据软件名称和业务描述生成菜单模块。

## 6. 生成逻辑

```text
读取母版 index.html
      ↓
处理软件名称和版本号
      ↓
读取风格预设和维度配置
      ↓
替换 title
      ↓
替换 h1 纯文字标题
      ↓
替换 tailwind.config 中的 colors、fontFamily、borderRadius
      ↓
替换 CSS 视觉变量
      ↓
替换 menuConfig
      ↓
替换右上角用户信息
      ↓
运行 9 条硬性要求校验
      ↓
校验通过则输出 index.html
```
