# IndexForge 硬性要求说明

本文件为 IndexForge 生成 index.html 时必须遵守的“红线规则”。第一组“维度”用于配置可变风格；本文件中的“要求”用于最终校验，强制要求不得被生成器破坏。

## R1 标题与品牌

顶栏标题只能是纯文字，旁边不能有任何 logo、图标、Emoji 或图片。

- 约束级别：强制
- 代码体现：`<header>` 中的 `<h1>` 标签只能输出纯文本字符串
- 禁止：`<img>`、`<i>`、`svg`、Emoji、FontAwesome 图标等出现在标题内部或标题紧邻位置作为品牌图形

## R2 Tailwind 配置

必须完整保留 `tailwind.config` 配置块。可以修改其中的颜色、圆角、字体等值，但不能删除该配置块。

- 约束级别：强制
- 代码体现：必须存在 `<script>tailwind.config = { ... }</script>`

## R3 子菜单展开方式

点击一级菜单后，子菜单必须在当前一级菜单项下方以内联方式展开，不能改成下拉、弹出、右侧悬浮、抽屉、侧滑或其他形式。

- 约束级别：强制
- 代码体现：`renderSubMenuInPlace()` 中必须保留 `activeMenuItem.insertAdjacentElement('afterend', wrapper)` 或等价写法

## R4 侧边栏位置

侧边栏必须在左侧，`app-sidebar` 或 `<aside>` 必须作为主内容 `flex` 容器中的第一个子元素。

- 约束级别：强制
- 代码体现：`.app-main` 内部先出现 `.app-sidebar`，再出现 `.app-content`

## R5 子页面不存在时的表现

当 iframe 加载不存在的 `.html` 文件时，界面必须保持空白干净，不能出现“404”、“页面建设中”、“暂无内容”、“开发中”等任何占位文字。

- 约束级别：强制
- 代码体现：`contentFrame.src = './${file}.html'`，不额外注入占位内容

## R6 主题切换器

每个 HTML 文件只能是一个固定主题，不能提供下拉菜单、按钮或状态逻辑来切换主题。

- 约束级别：强制
- 禁止：`<select>` 主题下拉、主题切换按钮、`currentTheme` 状态管理、多主题运行时切换

## R7 Tailwind 优先

布局、间距、颜色、圆角、阴影等尽量使用 Tailwind 工具类或母版已有类实现；仅滚动条、毛玻璃、复杂背景等少数情况使用自定义 CSS。

- 约束级别：强烈建议
- 建议：自定义 CSS 尽量可控，不要无限膨胀

## R8 核心 JS 逻辑不变

`menuConfig` 数据结构、`renderPrimaryMenu`、`renderSubMenuInPlace`、`navigateToSub`、`loadPage`、`window.navigateTo`、`message` 监听等核心功能逻辑不能被破坏。允许改视觉类名、菜单文案、文件名、图标映射、用户信息等。

- 约束级别：强制
- 目的：确保子菜单展开、iframe 跳转、跨 iframe 导航等功能稳定

## R9 用户名和头像可变化

右上角的用户名和头像属于装饰层，可以自由变化，包括改名、改头像图标、增加角色标签、增加修改密码/退出登录下拉菜单，或者完全移除。

- 约束级别：灵活
- 注意：用户区变化不能影响 R1 的标题纯文字要求

## 生成后红线检查清单

- [ ] 标题是否纯文字
- [ ] 是否保留 `tailwind.config`
- [ ] 子菜单是否内联插入在点击项下方
- [ ] 侧边栏是否在左侧
- [ ] iframe 是否没有额外占位文字
- [ ] 是否没有主题切换器
- [ ] 是否主要使用 Tailwind 或母版已有类
- [ ] 核心 JS 函数是否完整
- [ ] 用户信息变化是否符合预期
