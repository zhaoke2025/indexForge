# IndexForge 硬性要求说明

本文件为 IndexForge 生成 index.html 时必须遵守的“红线规则”。第一组“维度”用于配置可变风格；本文件中的“要求”用于最终校验，强制要求不得被生成器破坏。

## R1 标题与品牌

1、顶栏 h1 只能输出纯文字。顶栏标题只能是纯文字（必须是用户提供系统名称全称），旁边不能有任何 logo、图标、Emoji 或图片。

2、禁止在标题左侧放置收起侧边栏按钮，会被误认为 logo。

- 约束级别：强制
- 代码体现：`<header>` 中的 `<h1>` 标签只能输出纯文本字符串
- 禁止：`<img>`、`<i>`、`svg`、Emoji、FontAwesome 图标等出现在标题内部或标题紧邻位置作为品牌图形

## R2 Tailwind 配置

必须完整保留 `tailwind.config` 配置块。可以修改其中的颜色、圆角、字体等值，但不能删除该配置块。

- 约束级别：强制
- 代码体现：必须存在 `<script>tailwind.config = { ... }</script>`

## R3 子菜单（二级菜单）

1、子菜单必须插入到当前一级菜单项下方。点击一级菜单后，子菜单必须在下方以内联方式展开（插入到当前菜单项之后），不能改为下拉、弹出、侧滑或其他形式。

2、子菜单要比一级菜单靠右一些；在一级菜单有图标的情况下，至少和一级菜单的文字左对齐，不能比一级菜单靠左。

3、点击菜单时，禁止使用左侧边框高亮。

- 约束级别：强制
- 代码体现：`renderSubMenuInPlace()` 中必须保留 `activeMenuItem.insertAdjacentElement('afterend', wrapper)` 或等价写法

## R4 导航栏位置

导航菜单栏只能在左侧或顶部。除非额外要求，通常为左侧。

- 约束级别：强制
- 代码体现：左侧布局时，`.app-main` 内部先出现 `.app-sidebar`，再出现 `.app-content`；顶部布局时，导航栏位于主内容上方

## R5 空白 iframe

当 iframe 加载不存在的 `.html` 文件时，界面必须保持空白干净，不能出现“404”、“页面建设中”、“暂无内容”、“开发中”等任何占位文字。

- 约束级别：强制
- 代码体现：`contentFrame.src = './${file}.html'`，不额外注入占位内容

## R6 固定主题

每个 HTML 文件只能是一个固定主题，不能提供下拉菜单、按钮或状态逻辑来切换主题。

- 约束级别：强制
- 禁止：`<select>` 主题下拉、主题切换按钮、`currentTheme` 状态管理、多主题运行时切换

## R7 Tailwind 优先

布局、间距、颜色、圆角、阴影等尽量用 Tailwind 工具类实现，仅滚动条、毛玻璃等极少数情况使用自定义 CSS。

- 约束级别：强烈建议
- 建议：自定义 CSS 尽量可控，不要无限膨胀

## R8 核心 JS 逻辑

`menuConfig` 数据、`renderPrimaryMenu`、`renderSubMenu`、`navigateTo`、`message` 监听等核心功能逻辑不能改动，只能调整视觉样式类。

- 约束级别：强制
- 目的：确保子菜单展开、iframe 跳转、跨 iframe 导航等功能稳定

## R9 用户信息

用户名和头像可以自由变化（改名、改图标、加标签、甚至完全移除），不受限制。用户名长度遵循实体要求。

- 约束级别：灵活
- 注意：用户区变化不能影响 R1 的标题纯文字要求

## R10 禁止出现无关数据

无论是侧边栏还是顶栏，都禁止出现统计数字和版本号。

- 约束级别：强制
- 目的：避免生成与系统业务无关的统计信息或版本信息

## R11 退出系统

无论是在用户名头像下拉中退出登录，还是在顶栏最右侧或侧边栏最底部，都必须有退出登录的按钮或者文字。

- 约束级别：强制
- 目的：确保用户始终可以找到明确的退出入口

## 生成后红线检查清单

- [ ] 标题是否纯文字
- [ ] 是否保留 `tailwind.config`
- [ ] 子菜单是否内联插入在点击项下方
- [ ] 导航栏是否在左侧或顶部
- [ ] iframe 是否没有额外占位文字
- [ ] 是否没有主题切换器
- [ ] 是否主要使用 Tailwind 或母版已有类
- [ ] 核心 JS 函数是否完整
- [ ] 用户信息变化是否符合预期
- [ ] 是否没有无关统计数字和版本号
- [ ] 是否存在明确的退出系统入口
